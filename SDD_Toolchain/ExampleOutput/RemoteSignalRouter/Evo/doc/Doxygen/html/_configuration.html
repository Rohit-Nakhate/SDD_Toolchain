The module is taken its configuration from ComServFlex sst and Communication Matrix. The configuration tool is a batch file (RemoteSignalRouter_Generate.bat) which needs to be called from command line from the folder: mc_sw\appl\ComServer\RemoteSignalRouter\(VARIANT)\tools\ Any new PDUs or additional requirements can be configured in mc_sw\appl\ComServer\RemoteSignalRouter\(VARIANT)\cfg\rteLight\mirror\RemoteSignalRouter_PDUs.xml

The code generator uses the xsl programming language. It's inputs are available after a successfull RteLight generation.
For each generated file there is a templatate having the same name + the .xsl extension. For example RemoteSignalRouter_Routing.c.xsl will generate the RemoteSignalRouter_Routing.c source file.
